How to Host Your Flask App on PythonAnywhere for Free

In this article, I will guide you throughout the process of hosting your Flask Application live on PythonAnywhere for free.

Step 1: Create a requirements.txt

Create a new file in your project directory called requirements.txt. Make sure the spelling is correct. In the file write down all the libraries your package uses as well as the version. Here is how you can get the module version.

Here is what my requirements.txt looks like for my simple app.

flask==1.1.1

Step 2: Create a PythonAnywhere account

Head over to https://www.pythonanywhere.com/pricing and click on the button that says “Create a Beginner account.” This will allow you to create a free account. After clicking that, you will be asked to enter your email and to create a username and password. Make sure you choose your username wisely, because you will NOT be able to change it and because your website domain will be https://www.your-username.pythonanywhere.com. Make sure to agree to their “Terms and Conditions,” then click register.

Step 3: Configuration for your Web App

After you have created your account, you will be redirected to your dashboard. Near the top right corner of the dashboard, click on the “Web” link. To create a new web app, click on the blue “Add a new web app” button. Now you will have to follow a simple procedure to create your app.

Step 3.1: Your app’s domain name

Since you have created a beginner account, your domain name will be https://www.your-username.pythonanywhere.com. So you can go ahead and click “Next.” If you want a custom domain, you will have to upgrade your account.

Step 3.2: Select a Python Web framework

I’m guessing that since you are reading this article you are using Flask as your Python Web framework, so go ahead and click on “Flask.” Now choose the Python version for your app. I am going to go with the latest version yet, which is Python 3.8 (Flask 1.1.1).

Step 3.3: Quickstart new Flask project

In this step, you are just shown where the files for your flask app are going to be stored. You are given a default file called flask_app.py, but you can edit/remove it later. The path to your Flask App should look something like this.

/home/your-username/mysite/flask_app.py

Now you can click the “Next” button.

Your Web App Dashboard

Now you should get redirected to a new page which is your web app dashboard. This page has a lot of useful information about the statistics of your web app. At the top of your dashboard, where it says “Configuration for,” click the link right below it. This link is your website domain and after you click it, you will be taken to a new page which should show a message, “Hello from Flask!” If everything so far has worked out, then CONGRATS!

Step 4: Editing our default website

Now we have a simple live Flask app which just says “Hello from Flask!” Let’s change that and add our code instead. The first thing you will want to do is click the “Files” link on the top right corner of your dashboard. This will take you to the PythonAnywhere file manager, where you can navigate through folder and files. Under the directories category, you should see two folders, named .local/ and mysite/.

|--> /home/your-username/
     |--> .local/
     |--> mysite/

Click on mysite/ and inside that folder you will see a __pycache__/ folder and a flask_app.py file. Now let’s delete the flask_app.py file and add our own code.

Step 4.1: Deleting the default file

To delete the flask_app.py file, simply click on the trash icon right next to it and then you will be asked to confirm the deletion so just click “Yes” for the confirmation as well.

Step 4.2: Uploading code

To upload code, click on the yellow “Upload a file” button and choose the file you want to upload. You will only be able to upload files one at a time, so if you have multiple files, then you will have to repeat this step multiple times. Make sure to also upload your requirements.txt we made in Step 1. If you have a templates/ , static/ or any other type of directory, then don’t try uploading the directory or files from the directory yet. Here is what my main.py looks like.

Step 4.3: Creating a new directory

To create a new directory like templates/ or static or anything else, type the name of your directory in the entry that says “Enter new directory name.” Now click on “New directory.” You will immediately be directed into your new directory as you’ll notice by the path on the top.

/home/your-username/mysite/directory-name/

Now you can upload your HTML templates, static CSS files, or static images, by clicking on the “Upload a file button” just like we did in Step 4.2. This is what my index.html looks like.

After uploading all my files and creating the necessary directories for my app, this is what my layout looks like.

|--> /mysite/
     |--> main.py
     |--> requirements.txt
     |--> templates/
          |--> index.html
     |--> static/
          |--> favicon.png

Step 4.4 Installing the requirements.txt

Before reading the rest of this step, make sure you have already uploaded your requirements.txt in the mysite/directory. Now, to install the requirements.txt file we made in Step 1, navigate to your mysite/ directory and click on your root file. Your root file is the Python file you execute to run your application. For me, it would be main.py. When you click on the file, you will be at the PythonAnywhere Editor and you should be able to see the file contents. At the bottom of the editor, you should see a blue button that says “$ Bash console here.” Go ahead and click on it then wait for a few seconds for the console to load. Now install the requirements.txt file by running the following command in the Bash Console.

pip3 install -r requirements.txt --user

Step 5: Configuring the root file

If you remember the file path we were given in Step 3.3, we had a flask_app.py file, which was our root file. In other words it was the file that once executed, it would run our entire application. Now we have deleted that file and we need to tell PythonAnywhere which Python file it should execute to run the entire application. To do this, head over to your website dashboard by clicking on the “Web” navigation link on the top right corner. Once at your website dashboard, scroll down until you get to the “Code” section. Click on the third file link which is the WSGI configuration file. This is the file that tells PythonAnywhere what Python script to execute.

Step 5.1: Editing the import statement

Line 16 on the WSGI file should look like this.

from flask_app import app as application

The first thing you want to do is change “flask_app” to the name of your root file. Check out Step 4.4 to identify which script of yours is your root file. Since the name of my root file is main.py I’ll edit the import statement to this.

from main import app as application

The second thing you want to do is import your Flask(__name__) object from your root file. To do this, replace “app” with the name of the variable that stores that value Flask(__name__). Here are some examples.

Since I named my variable app I will leave my import statement as it is.

Step 5.2: Run the Web Application!

Now its finally time to run the web application! To do so, go back to your website dashboard and click the green button under the “Reload” category. Finally, click on the link to your website domain and CONGRATS!!!
